# AivisSpeech Studio — WebUI

[AIVIS Router](https://github.com/hit1023/me-aivis-router) のブラウザ操作用フロントエンドです。  
Nginx で配信するシンプルな単一HTMLファイル構成です。

---

## 機能

### 🎵 音声合成タブ
- スピーカー・スタイルの選択
- テキスト入力（最大500文字）
- 話速・音高・抑揚・音量・緩急・無音倍率のリアルタイム調整
- **アクセント可視化** — 音声生成後にアクセント句ごとのピッチ曲線をリアルタイム表示
- MP3再生 & ダウンロード
- VRAMロード状態の確認・強制アンロード
- **スピーカープリセット** — スピーカーごとにパラメータを保存・自動適用
- **モデルインストール** — `.aivmx` ファイルをドラッグ＆ドロップまたはファイル選択でアップロード
- **モデルアンインストール** — インストール済みモデルをワンクリックで削除（確認ダイアログ付き）
- **リアルタイムcurlサンプル** — 現在のパラメータ設定をそのままcurlコマンドで確認・コピー可能

### 📖 単語辞書タブ
- 固有名詞・読み方をユーザー辞書に登録
- **インタラクティブなアクセントビジュアライザー**（スライダーでピッチ曲線をリアルタイムプレビュー）
- **単語プレビュー再生** — 登録単語行の「▶ 再生」ボタンで即座に読み上げ確認
- ひらがな入力 → 自動カタカナ変換
- 品詞の正確な表示（人名・地名・組織名などを `part_of_speech_detail` から正しく判定）
- 登録単語の検索・編集・削除

### 📝 置換ルールタブ
- 音声合成前に適用するテキスト置換ルールを管理
- 英字固有名詞の誤読対策に有効（例: `Mumon → ミューモン`）
- ルールの追加・検索・削除（2ステップ確認式で誤削除を防止）
- **一括インポート** — `src@dst` 形式の `.txt` ファイルをドラッグ＆ドロップで一括 UPSERT
- **エクスポート** — 登録済みルールを `name.txt`（`src@dst` 形式）としてダウンロード

> **単語辞書 vs 置換ルール**  
> 英字の単語は辞書に登録するとカタカナに変換されてしまい、
> コンテキストによって効かないケースがあります。
> 英字固有名詞には置換ルールを使うのが確実です。

### 📜 履歴タブ
- 全 TTS 発話を新しい順に一覧表示
- 話者でフィルタリング可能
- ページネーション（件数切り替え対応）
- **📋 コピー** — 発話テキストをクリップボードにコピー
- **▶ 再生** — その履歴アイテムと同じ設定で音声を即時再生（再クリックで停止）

### 🔤 読みチェックタブ
- 単語リストをテキストエリアに入力（または `.txt` ファイルをインポート）してバッチ処理
- **AivisSpeech の実際の読みを自動取得** — `/audio_query` 経由でアクセント句を解析し、各単語のカナを自動表示
- **▶ 再生** ボタンで実際の音声を確認
- **✗ 修正** ボタンで読みが間違っていた場合のインライン修正パネルを表示
  - **置換ルールへ追加** — 表示された読みを元に置換ルール（`src → dst`）を1クリックで登録
  - **単語辞書へ** — 単語辞書タブに遷移し、対象単語・読みを自動入力した状態でフォームを開く
- 処理は3並列で実行し、プログレスバーでリアルタイム進捗表示

## 必要な環境

- Docker / Docker Compose
- [AIVIS Router](https://github.com/hit1023/me-aivis-router) が起動していること

---

## セットアップ

```bash
git clone <このリポジトリのURL>
cd AivisWebUI
docker compose up -d
```

ブラウザで `http://localhost:8888` を開きます。

---

## 接続先APIの変更

`.env` ファイルで `AIVIS_API_URL` を設定します。

```env
AIVIS_API_URL=https://your-aivis-router-url
```

コンテナ起動時に `docker-entrypoint.sh` が `index.html` 内のプレースホルダーを自動置換します。
